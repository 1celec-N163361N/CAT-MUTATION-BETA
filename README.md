# Mutation Portal - Professional Edition (Beta)

أداة اختبار اختراق احترافية مصممة لاختبار واجهات برمجة التطبيقات (APIs) وتطبيقات الويب من خلال إنشاء طفرات (Mutations) في طلبات HTTP ورصد أي استجابات شاذة (Anomalies) بشكل مباشر.

##  المميزات
- **محرك طفرات شامل**: يدعم فحص (HTTP Methods, Parameter Pollution, Header Injection, Content-Type, Path Fuzzing, JSON Structure وغيرها).
- **واجهة مستخدم احترافية**: تصميم داكن (Dark Mode) مناسب لبيئات العمل الاحترافية مع خلفية Silk متحركة وتأثيرات بصرية مميزة.
- **دعم اللغة العربية**: الواجهة تدعم الترجمة الفورية إلى اللغة العربية بضغطة زر.
- **سجلات حية (Live Logs)**: رصد مباشر للطلبات مع تحليل فوري للنتائج واكتشاف الشذوذ بناءً على حجم ونوع وحالة الاستجابة.
- **تعدد الخيوط (Multi-Threaded)**: إرسال طلبات متزامنة للتحقق السريع.

##  متطلبات التشغيل

الأداة مبرمجة باستخدام لغة Python ولا تتطلب سوى مكتبة واحدة إضافية.

1. تأكد من تثبيت Python 3 على جهازك.
2. قم بتثبيت مكتبة `requests` عبر الأمر التالي:
```bash
pip install requests
```

##  طريقة التشغيل

1. قم بتحميل ملف الأداة `cat_mutation_portal_beta.py`.
2. افتح موجه الأوامر (Terminal / Command Prompt) في نفس المسار الموجود فيه الملف.
3. قم بتشغيل السكربت باستخدام الأمر:
```bash
python3 cat_mutation_portal_beta.py
```
4. بمجرد التشغيل، ستظهر لك رسالة تفيد بأن السيرفر يعمل.
5. افتح متصفحك وقم بزيارة الرابط التالي:
[http://127.0.0.1:5050](http://127.0.0.1:5050)

##  كيفية الاستخدام

1. **Target Base URL**: أدخل الرابط الأساسي للهدف (مثال: `https://target.com`).
2. **Raw HTTP Request**: الصق طلب HTTP الخام الذي تريد فحصه (يمكنك أخذ هذا الطلب من أدوات مثل Burp Suite).
3. **الإعدادات**: 
   - **Threads**: عدد الخيوط المتزامنة لزيادة سرعة الفحص.
   - **Timeout (s)**: أقصى وقت للانتظار قبل إلغاء الطلب.
   - **Retries**: عدد مرات إعادة المحاولة في حال فشل الطلب.
4. **Mutation Categories**: حدد نوع الطفرات (الهجمات) التي تود تطبيقها.
5. اضغط على **"Execute Mutations"** لتبدأ الأداة في العمل.
6. تابع النتائج بشكل مباشر من قسم **Live Logs** أو انقر على **Live Response Monitor**.

##  نظرة على آلية عمل الكود (شرح مبسط)

- **استيراد المكتبات (Imports)**: الأداة تعتمد على مكتبات مدمجة في بايثون مثل `http.server` لإنشاء خادم الويب، و `threading` لتعدد المهام، بالإضافة لـ `requests` لإرسال طلبات HTTP.
- **إعدادات الأداة (Config)**: استخدام `dataclasses` لتخزين إعدادات الفحص كعدد الخيوط، وأوقات التأخير.
- **دوال الطفرات (Mutators)**: دوال مثل `mutate_methods`، `mutate_parameter_pollution` تأخذ الطلب الأصلي وتقوم بتعديل أجزاء معينة فيه (مثل الهيدر، أو الرابط، أو الجسم) لتوليد مئات الطلبات الخبيثة المحتملة.
- **البصمة الرقمية (Fingerprinting)**: دالة `response_fingerprint` لتحليل استجابة الخادم وتحديد حالتها وحجمها ومحتواها. دالة `is_anomaly` تقارن هذه البصمة بالاستجابة الطبيعية وتحدد إذا ما كان هناك خلل في الخادم (مما قد يشير لثغرة).
- **إرسال الطلبات المتزامنة**: استخدام نظام `threading` في بايثون لإنشاء عدة عمال (Workers) يسحبون الطلبات من قائمة المهام وينفذونها في نفس الوقت لتقليل مدة الفحص.
- **واجهة المستخدم المدمجة**: الكود يتضمن قوالب HTML/CSS/JS (داخل المتغير `PAGE_TEMPLATE` والمتغيرات المرتبطة به) للواجهة الرسومية بحيث لا تحتاج الأداة إلى أي ملفات خارجية لعرض الواجهة الاحترافية (تشمل الترجمة للعربية وتأثيرات الخلفية).
